# 产品规划文档

## 一、产品定位

实时虚拟人开放平台是一个AI驱动的SaaS产品，让企业和个人能够快速创建、训练和部署专属虚拟人，应用于客服、直播、教育等场景。

## 二、目标用户与痛点

### 核心用户群

1. **中小企业**
   - 电商：需要24小时直播带货
   - 客服中心：降低人力成本
   - 在线教育：标准化课程讲解

2. **内容创作者**
   - 短视频博主：提高内容产出效率
   - 知识IP：将知识产品化

3. **开发者**
   - 需要集成虚拟人能力的应用

### 核心痛点

1. **成本高**：真人主播/客服时薪成本高
2. **效率低**：内容制作周期长
3. **门槛高**：现有虚拟人方案需要专业团队
4. **无法实时交互**：录播方案体验差

### 解决方案

- 低代码平台，5分钟创建虚拟人
- 7×24小时工作，成本降低80%
- 基于RAG的实时对话，智能化程度高
- 按需付费，降低使用门槛

## 三、核心特性：Multi-Agent协作

### 3.1 单虚拟人 vs 虚拟人团队

**单虚拟人模式**：
- 一个虚拟人独立处理所有问题
- 适合简单场景（如单一产品客服）

**Multi-Agent团队模式**：
- 多个虚拟人组成专业团队
- 根据问题类型自动路由和转接
- 适合复杂场景（如企业客服中心、在线教育）

### 3.2 应用场景示例

**客服场景**：
```
用户咨询 → 接待员Agent（初步分类）
  ├─ 产品问题 → 产品专家Agent
  ├─ 技术问题 → 技术支持Agent
  └─ 售后问题 → 售后服务Agent
```

**教育场景**：
```
学生提问 → 讲师Agent（主讲）
  ├─ 基础问题 → 助教Agent
  ├─ 作业批改 → 批改Agent
  └─ 答疑 → 答疑Agent
```

## 四、MVP功能列表

### 4.1 虚拟人创建

**功能描述**：用户可以创建个性化虚拟人

**核心能力**：
- 形象选择：6种预设模板（商务男/女、教师、客服等）
- 人设配置：名称、性格、说话风格
- 声音配置：选择音色（温柔、专业、活力等）
- 保存管理：支持创建多个虚拟人

**技术实现**：
- 前端表单 + 实时预览
- 后端存储配置到SQLite

### 3.2 知识库管理

**功能描述**：上传文档构建虚拟人的专业知识

**核心能力**：
- 文档上传：支持TXT、PDF格式
- 自动处理：文本提取、分块、向量化
- 知识列表：查看已上传的文档
- 删除管理：移除不需要的知识

**技术实现**：
- 文档解析：PyPDF2/pdfplumber
- 文本分块：按段落或固定长度
- 向量化：使用embedding模型
- 存储：ChromaDB向量数据库

### 3.3 实时对话

**功能描述**：与虚拟人进行智能对话

**核心能力**：
- 文本对话：输入问题，获得回答
- 知识检索：基于RAG从知识库检索相关内容
- 上下文记忆：记住对话历史
- 流式输出：逐字显示回答（可选）

**技术实现**：
- RAG流程：问题 → 向量检索 → 拼接prompt → LLM生成
- DeepSeek API调用
- WebSocket实时通信（可选）

### 4.4 数据看板

**功能描述**：查看使用统计

**核心能力**：
- 对话统计：总对话数、今日对话数
- 知识库统计：文档数量、总字数
- 虚拟人列表：已创建的虚拟人数量
- 简单图表：对话趋势（可选）

**技术实现**：
- 数据统计SQL查询
- 前端图表库（Recharts）

### 4.5 虚拟人团队（Multi-Agent）

**功能描述**：创建虚拟人团队，实现多Agent协作

**核心能力**：
- 团队创建：选择多个虚拟人组成团队
- 角色定义：主接待、专家、辅助等角色
- 路由规则：根据关键词或意图自动分配
- 转接逻辑：虚拟人之间的对话转接

**技术实现**：
- 意图识别：使用LLM判断问题类型
- 路由引擎：根据规则分配到对应虚拟人
- 上下文传递：转接时保持对话历史

**场景模板**：
- 客服团队（接待+技术+售后）
- 教育团队（讲师+助教+答疑）

## 五、技术架构

### 4.1 前端架构

```
React App
├── pages/           # 页面
│   ├── Dashboard    # 工作台
│   ├── Create       # 创建虚拟人
│   ├── Knowledge    # 知识库
│   ├── Chat         # 对话
│   └── Analytics    # 数据
├── components/      # 组件
├── store/           # 状态管理
└── api/             # API调用
```

### 4.2 后端架构

```
FastAPI
├── routers/         # 路由
│   ├── avatar       # 虚拟人管理
│   ├── knowledge    # 知识库
│   ├── chat         # 对话
│   └── analytics    # 数据统计
├── services/        # 业务逻辑
├── models/          # 数据模型
└── utils/           # 工具函数
```

### 4.3 数据流

1. **创建虚拟人**：前端表单 → API → SQLite存储
2. **上传知识**：文件 → 解析 → 向量化 → ChromaDB
3. **对话流程**：问题 → 向量检索 → 拼接上下文 → DeepSeek → 回答

## 五、涉及的AI技术

### 5.1 自然语言处理

- **大语言模型**：DeepSeek（对话生成）
- **Embedding模型**：文本向量化（可用OpenAI或本地模型）
- **RAG技术**：检索增强生成

### 5.2 语音技术（可选）

- **TTS**：Web Speech API或讯飞语音
- **语音克隆**：后续可接入

### 5.3 向量数据库

- **ChromaDB**：本地向量存储和检索
- **相似度搜索**：余弦相似度
